2. Summary

PocketCoach does not collect any personal data. The app runs offline in normal use. Optionally, the ball library can be updated from a public GitHub repository by tapping a button — this transmits your IP address to GitHub (as does any web request). All your input (player profiles, balls, oil patterns, throws) is stored exclusively on your device.

3. Local Data Storage

PocketCoach stores the following data locally in the app's internal storage (technically: localStorage and IndexedDB of the Android WebView):

  • Player profiles (self-chosen name, skill settings)
  • Entered releases (rotation axis, tilt, rotation, etc.)
  • Stored balls, layouts and bag assignments
  • Custom and preset oil patterns
  • UI settings (language, card visibility, theme)
  • Most recent simulated throws and lane state
  • Ball library cache (mirrored state of the optional online repository)
  • Downloaded ball photos (for offline display in the IndexedDB cache)

This data never leaves your device. It is not transmitted to the developer or to any third party.

4. File Access

When you import a custom oil pattern, PocketCoach uses the standard Android file picker to ask you for a single file (PDF, JSON or CSV). The file is processed locally; its content is added to the app's own data. The app does not access any other files on your device and does not require special file permissions.

5. Optional Online Access (Ball Library)

Since version 1.4.0, the ball library can be updated from a public GitHub repository by tapping a button in the library picker. By default the app runs offline — the update only happens when you press the "Update" button.

Where does the request go?

  • raw.githubusercontent.com (GitHub Inc., USA)
  • Paths: /BeckerApps/pocketcoach-assets/main/balls.json (data) and /main/balls/<key>.webp (ball photos)
  • GitHub privacy: docs.github.com/en/site-policy/privacy-policies/github-general-privacy-statement

What is transmitted?

  • Technically required: your IP address, the User-Agent (standard Android WebView identifier) and the request timestamp — this is transmitted by any web request.
  • Not transmitted: no device identifier, no usage data, no personal data, no cookies, no tracking IDs.

Legal basis: Art. 6 (1)(a) GDPR — your consent through active confirmation of the "Update" button.

Note: The downloaded data is stored locally (see section 3), so the library works offline again afterwards.

6. Permissions

PocketCoach requests the Android INTERNET permission. It is used exclusively for the optional ball library update (see section 5). No automatic internet connection takes place. In particular, the app uses none of the following:

  • Location access
  • Camera or microphone access
  • Contacts, calendar or phone data

7. Cookies, Tracking, Advertising

PocketCoach uses no cookies, no tracking, no analytics SDKs (e.g. Google Analytics, Firebase) and no ad networks.

8. Third-Party Components

The app bundles the pdf.js library (Mozilla, Apache License 2.0). It is used exclusively offline for reading imported PDF files and does not communicate with the internet.

9. Google Play Store

This app is distributed through the Google Play Store. When you download and use the Play Store, Google may collect data (e.g. device ID, installation timestamp, crash reports). This data collection is performed by Google and is outside the control of the PocketCoach developer. See Google's privacy notice at: policies.google.com/privacy

10. Your Rights (GDPR Art. 15 ff.)

Since PocketCoach does not collect or process any personal data, the developer holds no personal data of yours that could be subject to rights of access, rectification or deletion.

You can delete your locally stored app data at any time by:

  • Removing profiles / data manually in the app settings, or
  • Resetting the app data via Android system settings: "Apps → PocketCoach → Storage → Clear data", or
  • Uninstalling the app (this removes all local data).
